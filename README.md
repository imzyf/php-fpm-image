## Origin

Three images are mirrored unmodified from [laradock/laradock](https://github.com/laradock/laradock), each into its own top-level directory:

- `php-fpm/` — from laradock's `php-fpm/` (generic `Dockerfile`, parameterized by `LARADOCK_PHP_VERSION`)
- `php-worker/` — from laradock's `php-worker/`
- `workspace/` — from laradock's `workspace/`

## Sync from upstream

```bash
bin/sync-upstream.sh   # pull latest php-fpm/, php-worker/, workspace/ from laradock/laradock
bin/generate-env.sh    # regenerate .env.laradock from all three Dockerfiles' ARGs
```

`.env.laradock` is a single root file with every `ARG` from all three Dockerfiles, grouped under a `# ===== <image> =====` heading per image (a name shared across Dockerfiles is only listed once). `.env.laradock.preference` holds this project's chosen overrides on top of those defaults — it's hand-maintained, not regenerated by `bin/generate-env.sh`.

After syncing, diff `.env.laradock` against its previous version to pick up any new/removed ARGs, and update `.env.laradock.preference` if a new ARG should be overridden.

## Build locally

```bash
make build                          # IMAGE_NAME=php-fpm PHP_VERSION=8.5
make build IMAGE_NAME=php-worker
make build IMAGE_NAME=workspace
make test                           # build + smoke-check (php -v, and pgsql/redis when IMAGE_NAME=php-fpm)
```

Build args come from `.env.laradock` merged with `.env.laradock.preference` (the latter wins on conflicting names).

## CI

`.github/workflows/dockerimage.yml` (workflow name `laradock-image`) builds PHP `8.5` for `linux/amd64,linux/arm64`, across all three images (`php-fpm`, `php-worker`, `workspace`), using the same `.env.laradock` + `.env.laradock.preference` merge as the Makefile for build-args.

Triggers: manual only, via the "Run workflow" button (`workflow_dispatch`) on the [Actions tab](https://github.com/imzyf/php-fpm-image/actions/workflows/dockerimage.yml).
